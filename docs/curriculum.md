# 🗺️ 바이브 코딩 캠프 전체 커리큘럼

코드를 하나씩 치는 과거의 방식은 잊으십시오. 이 120일(6개월) 코스는 코드를 "읽고, 붕괴를 예측하며, 프롬프트로 AI를 지휘하는" 오케스트레이션(디렉팅) 능력 배양에 모든 것을 걸었습니다.

---

## 🟢 Month 1: C# 해부학과 아키텍처 베이스 (진행 중)
**"AI가 오타 하나로 프로젝트를 날려 먹는 원흉 파악하기"**
*   **Week 1:** 데이터의 본질 (타입 안정성, 구조체 최적화, 참조 슬롯)
*   **Week 2:** 행동 강령과 생산 라인 (함수의 파라미터와 리턴값 해부)
*   **Week 3:** 흐름의 통제 (무한 루프와 쿠소게를 유발하는 조건문 박살 내기)
*   **Week 4:** 공장 짓기 (클래스 캡슐화와 객체 지향의 본질)

## 🔒 Month 2: 유니티 엔진과 OOP의 결합
**"아티스트의 에디터 통제권을 되찾고, 생명 주기를 지배하기"**
*   **Week 1:** 생명 주기의 감별사 (`Start` vs `Update` 렉 유발 원인 파악)
*   **Week 2:** 인스펙터의 지배자 (하드코딩을 부수고 `[SerializeField]`로 권한 뺏기)
*   **Week 3:** 컴포넌트 통신망 (`GetComponent` 이름 탐색 의존성 끊어내기)
*   **Week 4:** Update 커플링 탈출 미션 (여러 로직이 엉겨 붙은 신(God) 클래스 분리)

## 🔒 Month 3: 콘솔, 그리고 CLI 오케스트레이션
**"GUI 마우스를 던져버리고 시스템 전체 생태계를 텍스트로 지배하기"**
*   **Week 1:** 터미널의 권력 (CLI 기초 경로 및 디렉토리 제어)
*   **Week 2:** 명령어 대행자 (AI 에이전트의 위험 명령어 SafeToAutoRun 통제법)
*   **Week 3:** 로그(Log) 독해법 (시뻘건 Stack Trace에서 꼬리를 자르고 원인 찾기)
*   **Week 4:** 블랙박스 빌드 에러 환경 변수 탈출 미션

## 🔒 Month 4: 아키텍터 모드 - 싱글톤과 동기화
**"UI와 로직을 이혼시켜 단일 진실 공급원(SSOT) 규칙 세우기 (장애물 피하기 & 슈팅 튜닝)"**
*   **Week 1:** 관심사 분리 (SoC: UI와 데이터 모델의 분리 설계)
*   **Week 2:** 싱글톤(Singleton) 패턴의 폭주 막기 (유니티 매니저 병목 잡기)
*   **Week 3:** Invalid Sync (동기화 붕괴) 거짓말 데이터 추적 기법
*   **Week 4:** 가짜 물약 포션 버그 (단방향 데이터 플로우 아키텍처링)

## 🔒 Month 5: 메모리와 라이프사이클 통제
**"스마트폰을 녹여버리는 메모리 릭(Leak)과 좀비 객체 적발하기"**
*   **Week 1:** 가비지 컬렉션(GC) 폭발 방지와 인스턴시 데이터 캐싱
*   **Week 2:** 좀비 참견쟁이 (이벤트 리스너 구독 `+=` 과 해제 `-=` 구조 한 줄로 강제법)
*   **Week 3:** 침묵하는 에러 (AI의 `try-catch` 남용을 막고 Fail-fast 터트리기)
*   **Week 4:** 씬(Scene) 전환 시 쌓이는 잔여 찌꺼기 메모리 릭 폭파 미션

## 🔒 Month 6: 장르 특화 실전 오케스트레이션
**"나도코딩 실전 장르들의 핵심 아키텍처 뼈대를 구상하고 지시하기"**
*   **Week 1:** 플랫포머 아키텍처 (Raycast 레이더망과 상태 머신 패턴)
*   **Week 2:** 매치 3 퍼즐 아키텍처 (2D 배열 데이터 매니저와 시각적 코루틴 콜백 분리)
*   **Week 3:** 서바이버 로그라이트 아키텍처 (오브젝트 풀링과 화면 스크롤 최적화)
*   **Week 4:** Final Test - 프롬프트만으로 단일 진실 미니 게임 띄우기
