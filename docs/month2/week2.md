### 📅 [Day 1] 인스펙터 인테리어 - 정리 정돈의 미학

#### 📖 1. 예습 (쓰레기장 vs 최신식 조종석)
> **🚀 전투기 조종석의 난잡한 버튼들**
> 
> Month 1 파트에서 `[SerializeField]`를 달아주면 프라이버시를 지키면서 유니티 에디터 화면에 수정할 수 있는 구멍(슬롯)이 뚫린다고 배웠습니다. 
> 그런데! 변수가 1~2개일 땐 괜찮지만, 체력, 마나, 스피드, 점프력, 총알 사운드, 이펙트 프리팹 등 50개의 변수를 무지성으로 뚫어놓으면 어떻게 될까요?
> 스크립트 인스펙터가 마치 쓰레기장처럼 변해서, 아티스트가 총소리를 바꾸려다 실수로 체력을 지워버리는 대참사가 매일 터집니다.

#### ✅ 2. 모범 답안 (라벨지와 띄어쓰기)
AI에게 무조건 **이 세 가지 데코레이션 태그**를 필수적으로 적용하라고 프롬프팅 해야 합니다.

*   `[Header("라벨 이름")]`: 인스펙터 굵은 제목 카테고리 표시
*   `[Space(10)]`: 위아래 블록 여백(띄어쓰기) 주기
*   `[Tooltip("설명")]`: 마우스를 올렸을 때 말풍선 설명(가이드) 띄우기

```csharp
public class PlayerController : MonoBehaviour 
{
    // [문장 독해] 전투기 패널에 "스테이터스"라는 굵은 메뉴 이름판을 단다!
    [Header("⚔️ 전투 스테이터스")] 
    [Tooltip("캐릭터가 받는 최대 체력 세팅입니다.")]
    [SerializeField] private float maxHealth = 100f;
    [SerializeField] private float attackPower = 20f;

    // [문장 독해] 위쪽 메뉴랑 너무 겹치니까 15칸짜리 공백을 줘서 공간 분리를 해!
    [Space(15)] 
    
    [Header("🔊 사운드 파츠")]
    [SerializeField] private AudioClip attackSound;
}
```
**💡 디렉터의 쾌감**
이 3개를 강제로 박아 넣으면, 유니티 인스펙터가 흡사 최신 대작 게임의 플러그인처럼 **깔끔한 폴더별 라벨링**으로 나뉘어져 나타납니다. 밥 시켜 먹을 맛이 납니다!

### 📅 [Day 2] 슬라이더 록타이트 - Range와 캡슐화 

#### 📖 1. 예습 (뒷걸음질 치는 용사)
> **🔒 스피드 -100을 넣는 호기심 많은 기획자**
> 
> 기획자나 아트 디자이너는 때로 창의적인 파괴자입니다. 이동 속도를 기입하는 `[SerializeField] float speed;` 란에 심심해서 `-100`을 입력하고 플레이 버튼을 누르면, 용사가 뒤로 미끄러지는 문워크를 시전합니다! 이따위 기행을 막는 가장 직관적인 디바이더 락, 바로 **`[Range]`** 입니다.

#### ✅ 2. Range (슬라이더 고정)
```csharp
public class CarModule : MonoBehaviour 
{
    // 최솟값 0, 최댓값 200으로 강제 바리게이트를 쳐버리는 Range 마법!
    [Range(0f, 200f)] 
    [SerializeField] private float carSpeed = 50f;
}
```
유니티 화면에서 숫자 적는 칸이 **드래그 가능한 깔끔한 UI 슬라이더 바**로 변신하며, 아티스트가 죽었다 깨어나도 0 ~ 200 밖의 수치를 넣지 못하게 차단합니다!

#### 🗣️ 3. 지휘관의 프롬프팅
> **"야 AI, 너 캐릭터 이동 속도 변수에 왜 양수만 들어가게 바리게이트 안 쳐놨어? 기획자가 음수 넣어서 버그 터지는 거 구경할래? 당장 값 제한 필요한 건 `[Range(min, max)]` 먹여서 에디터 슬라이더로 만들어 놔!"**

### 📅 [Day 3] Property의 마술 - 데이터 은닉과 실시간 디버거

#### 📖 1. 예습 (자물쇠 채워진 유리 진열장 보여주기)
> **👀 은닉된 데이터(Private)를 유니티에서 훔쳐보고 싶을 때!**
> 
> `[SerializeField]`는 에디터에서 볼 수 있고 수정도 가능케 합니다. 그런데, "나는 게임 중에 코드가 실시간으로 체력을 깎는 걸 인스펙터로 눈팅만 하고 싶고, **마우스로 직접 조작하는 건 절대 막고 싶어!**" 하는 데이터가 있습니다. (예: 현재 계산된 누적 데미지량)

이때는 AI에게 유니티 패키지의 **Property 속성 표시기(또는 ReadOnly 커스텀)**를 만들어 달라고 지시하거나 단축키(Debug Mode)를 활용하는 방법을 씁니다. 지금은 디렉터로서 **"수정은 못 하고 눈으로 모니터링만 하는 데이터 표시"**를 당당하게 요구할 수 있다는 권력만 기억하십시오.

### 📅 [Day 4] ScriptableObject - 10만 대군 메모리 다이어트

#### 📖 1. 예습 (종이 공책표 vs 단단한 플라스틱 주민등록증)
> **💳 붕어빵의 영양성분 복사 비극**
> 
> 일반적인 `[SerializeField]`나 클래스 방식으로 몬스터 프리팹을 만들고 1,000마리를 생성하면? 
> 체력 100, 스피드 5라는 똑같은 글씨가 적힌 엑셀 데이터 테이블을 1,000명의 몬스터 메모리 지갑마다 **각각 무식하게 복사해서 붙여넣습니다!** (메모리 1,000배 낭비)
> 
> 하지만 몬스터의 최대 체력, 공격력 수치는 게임 중 잘 안 변하는 '원형 도감 정보'죠?
> 이걸 **플라스틱 카드(ScriptableObject)**로 1장만 구워서 꽂아 쓰면, 1,000마리의 몬스터가 1장의 공용 카드 정보를 눈으로 읽기만 합니다! 메모리 소모가 극단적으로 사라집니다.

#### ✅ 2. 카드 발급기와 사용법 (ScriptableObject)

**1단계: 신분증(카드) 공장 발급기 설계도 짜기**
```csharp
using UnityEngine;

// 에디터 우클릭 메뉴에 "몬스터 도감 카드 만들기" 버튼을 추가하라는 마법의 명령어!
[CreateAssetMenu(fileName = "NewMonsterInfo", menuName = "데이터/몬스터 스탯")]
public class MonsterStaticData : ScriptableObject 
{
    // 이 데이터들은 한 번 세팅하면 게임 중에 바뀌지 않는 고정값들입니다!
    [Header("기본 정보")]
    public string monsterName;
    public float maxHp;
    public float attackPower;
}
```

**2단계: 몬스터 스크립트에 플라스틱 카드(SO) 긁는 단말기 달아주기**
```csharp
public class MonsterEntity : MonoBehaviour 
{
    // 방금 구워낸 플라스틱 카드(SO)를 꽂아 넣을 빈 깡통 슬롯 1개!
    [SerializeField] private MonsterStaticData myStatCard;

    private float currentHp; // 실시간으로 깎이는 피통은 개인지갑에 별도 생성.

    void Start() {
        // 태어날 때 공용 플라스틱 카드에서 최대 체력을 읽어와 내 개인 피통에 붓는다!
        currentHp = myStatCard.maxHp; 
        Debug.Log($"나는 {myStatCard.monsterName}! 영원불멸의 SO 카드 능력치가 주입되었다!");
    }
}
```

#### 🗣️ 3. 지휘관의 프롬프팅
> **"야 AI, 몬스터 기본 공격력, 이동 속도, 무기 이름같이 절대 안 변하는 원본 베이스 데이터를 왜 클래스 안에 다 냅둬서 찍어낼 때마다 메모리 폭발하게 둬? 영원 불변하는 메타(Meta) 데이터들은 싸그리 `ScriptableObject(SO)` 파일로 뽑아내서 데이터 통폐합시켜!"**

### 📅 [Day 5] 🕹️ 미니 프로젝트 & 에디터 인테리어 방탈출

#### 🎮 [미니 프로젝트] 플라스틱 신분증(SO) 갈아 끼우기
1.  **발급소 생성:** 위 `MonsterStaticData` 코드를 복사해 유니티에 스크립트를 만듭니다. (절대 컴포넌트로 안 붙습니다!)
2.  **카드 생성:** Project 폴더 창(하단)에서 마우스 우클릭 -> `Create` -> `데이터` -> `몬스터 스탯` 을 눌러 **플라스틱 카드 파일 2장**을 생성합니다.
    *   첫 번째 카드 이름: `GoblinCard` (인스펙터에 가서 name: 고블린, hp: 10 입력)
    *   두 번째 카드 이름: `DragonCard` (name: 화룡, hp: 9999 입력)
3.  **장착:** 유니티 큐브에 달아둔 `MonsterEntity` 스크립트 슬롯(`myStatCard`)에 `GoblinCard` 파일을 마우스로 쭉 끌어서(드래그 앤 드롭) 장착하고 플레이! -> 콘솔에 고블린 메시지 출력 확정.
4.  **카트리지 체인지:** 플레이 종료 후 큐브 슬롯에 이번엔 `DragonCard`를 갈아 끼우고 플레이! 코드를 한 줄도 안 바꿨는데 최강의 화룡으로 둔갑하는 아키텍처의 쾌감이 쏟아집니다!!
