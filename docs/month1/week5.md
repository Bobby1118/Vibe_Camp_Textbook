### 📅 [Day 1] 외계어 해독: 스크립트 맨 위의 2가지 비밀

유니티에서 아무것도 없는 새 큐브(Cube)를 만들고, `PlayerObject`라는 C# 스크립트를 만들어 더블클릭 하면 무분별하게 적혀있는 첫 세 줄이 보입니다.

```csharp
using UnityEngine; // [비밀 1]

public class PlayerObject : MonoBehaviour // [비밀 2]
{

}
```

#### 🛠️ [비밀 1] `using UnityEngine;` 의 정체 (도구 상자 수입하기)
> **🧰 "옆집 유니티 본사에서 공구 상자 좀 빌려 쓸게!"**
> 
> 컴퓨터 백지상태에서는 `GameObject`, `Vector3`, `Transform` 같은 유니티 전용 단어를 모릅니다. 치면 그냥 빨간 줄 에러가 납니다.
> 코드 맨 위 꼭대기에 달린 `using UnityEngine;` 은 마법의 수입 신고서입니다. **"유니티 엔진 본사에서 만든 방대한 요리 도구 상자(라이브러리) 전체를 지금 내 스크립트 도마 위로 쏟아부어 줘!"** 라는 뜻입니다. 
> 이 한 줄이 있어야만 우리가 그토록 쓰고 싶어 하는 유니티의 화려한 기능들을 자유롭게 갖다 쓸 수 있습니다.

#### 🛠️ [비밀 2] `: MonoBehaviour` 의 정체 (유니티 정식 공무원증 발급)
> **🏷️ "이 스크립트는 단순 메모장이 아니라, 부품입니다!"**
> 
> 코드를 아무리 기깔나게 짜도 클래스 이름 옆에 `: MonoBehaviour` (모노비헤이비어) 꼬리표를 달아주지 않으면, 유니티 에디터 화면에서 마우스로 아무리 큐브에 드래그해도 스크립트가 붙지 않고 튕겨 나옵니다.
> 저 외계어는 유니티 엔진 시스템에게 **"내 스크립트는 이제부터 드래그 앤 드롭으로 장착할 수 있는 유니티 정식 부품(Component)이다!"라고 공무원 목걸이(자격증)를 채워주는 행위**입니다! 
> *(디렉터님, 이제는 저 길고 복잡한 이름에 쫄 필요가 1%도 없습니다. 그저 끈적이 테이프일 뿐입니다!)*

### 📅 [Day 2] 실전 적용: 에디터 화면에 냅다 구멍 뚫기

이제 우리가 1주 차에 배운 변수 제어법이 실제로 화면에 어떤 마법을 부리는지 볼까요?

```csharp
using UnityEngine;

public class PlayerObject : MonoBehaviour
{
    // 1번 방법: 무지성 대문 개방 (해킹의 위험이 있음)
    public int publicHp = 100;

    // 2번 방법: 완전 자물쇠 철통 보안 (화면에 안 보임)
    private int privateHp = 100;

    // 3번 방법: 보안을 지키면서 유니티 화면에만 뚫어주기 (디렉터의 주력 기술!)
    [SerializeField] private int safeHp = 100;
}
```

#### 👀 유니티 엔진으로 돌아가서 큐브를 클릭해 봅시다! (마법의 발생)

스크립트를 저장(Ctrl+S)하고 유니티 에디터로 돌아와서 큐브의 **인스펙터 창(Inspector)**을 보면 믿기 힘든 광경이 벌어집니다!

1.  **`public int publicHp`** 
    👉 인스펙터 창에 **`Public Hp [ 100 ]` 이라는 숫자를 바꿀 수 있는 입력 칸이 물리적으로 뿅 하고 뚫렸습니다!!** 이래서 프로그래머들이 귀찮으면 죄다 `public`을 남발해서 화면에 열어두는 겁니다. 
2.  **`private int privateHp`** 
    👉 에디터 화면에 **아예 보이지조차 않습니다.** 디자이너가 숫자를 바꾸고 싶어도 바꿀 구멍이 없습니다. 무시무시한 자물쇠죠!
3.  **`[SerializeField] private int safeHp`** 
    👉 `private`의 성질을 가져서 남들 코드에선 해킹(수정)을 못 하게 막아버렸는데, **놀랍게도 유니티 에디터 화면에는 `Safe Hp [ 100 ]` 이라고 마음대로 수치를 조작할 수 있는 구멍만 기적처럼 열려 있습니다!** 

🔥 **디렉터의 성취:**
디자이너님이 기획서에 `[SerializeField] private`으로 열어두라고 호통치면, 프로그래머는 코드를 안전하게 보호하면서도 동시에 디자이너가 마우스로 조작할 수 있는 **[에디터 슬롯]**을 배달해 바치게 됩니다. 코드와 UI 화면이 대화하는 순간입니다!

### 📅 [Day 3] 실전 적용: 1달 치 노하우를 엔진으로 찌르기

이제 유니티 에디터 화면의 텍스트가 어떻게 보이는지 1달 치 배운 내용으로 테스트해보십시오!

```csharp
using UnityEngine;

public enum AnimalType { Dog, Cat, Bird } // 나만의 메뉴판

public class ZooManager : MonoBehaviour // 유니티 부품 자격증 달기!
{
    [Header("에디터에 제목 글씨를 띄우는 마법")] // 이것도 화면(UI)과 코드가 대화하는 기술입니다!
    [SerializeField] private float speed = 5.5f; // 아티스트가 화면에서 속도를 조절할 구멍
    
    [SerializeField] private AnimalType selectAnimal; // 화면에서 드롭다운 메뉴로 고를 수 있는 구멍!
}
```

**🎬 눈앞에서 벌어지는 유니티 매직 쇼!**
위 코드를 저장하고 유니티를 보면, 인스펙터 화면이 이렇게 예쁘게 변합니다.
> **에디터에 제목 글씨를 띄우는 마법** (굵은 회색 글씨 띠가 생김)
> Speed  [ 5.5 ]
> Select Animal  [ Dog ▼ ] *(클릭하면 Dog, Cat, Bird가 드롭다운 바이브 메뉴판으로 나옴! 대문자 오타 낼 걱정 0%!!)*
