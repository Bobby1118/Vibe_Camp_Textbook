## Week 3: 🚦 흐름의 통제 (조건문과 반복문)

지금까지 우리는 몬스터를 만들고(`Class`), 몬스터에게 동작(`Method`)을 지시했습니다. 하지만 모든 몬스터가 똑같이 무지성 돌진만 하면 재미가 없죠! 3주 차에서는 상황에 따라 행동을 바꾸는 **'조건문'**과 노가다를 없애주는 **'반복문'**을 완벽하게 통제하는 법을 배웁니다.

### 📅 [Day 1] if ~ else (조건문) - 나이트클럽의 기도

#### 📖 1. 예습 (진정한 가드맨 비유)
> **🛡️ "민증 안 가져왔으면 집에 가세요!"**
> 
> 컴퓨터는 시키는 대로 무식하게 처음부터 끝까지 코드를 쭉 읽어내려 갑니다. 그래서 디렉터는 길목 중간중간에 **'기도(Bouncer)'**를 세워둬야 합니다.
> "현재 플레이어 체력이 0인가?(`if`) 맞으면 게임 오버 화면 띄워! 아니면(`else`) 그냥 하던 플레이 마저 하게 들여보내 줘."
> 이것이 바로 흐름을 가르는 기준점, **`if-else` 블록**입니다.

#### ✅ 2. 모범 답안과 외계어 독해
AI가 체력 검사 코드를 짜면 이렇게 짭니다. 코드를 읽어봅시다.

```csharp
public void TakeDamage(int damage) 
{
    hp -= damage; // 일단 체력을 깎는다

    // [문장 1 독해] 나이트클럽 기도의 체력 검사
    if (hp <= 0) 
    {
        Debug.Log("플레이어 사망!");
        GameOver(); // 피가 0 이하면 게임 끝!
    } 
    else 
    {
        Debug.Log("아직 안 죽었음. 계속 싸우셈.");
    }
}
```

**🔍 잠깐! 코드 외계어 독해 가이드**
*   👉 **코드 공식:** `if (조건) { 맞으면 이거실행 } else { 틀리면 이거실행 }`
*   🗣️ **소리내어 읽기:** *"만약(`if`) 남은 체력 수치가 0보다 작거나 같다면(`<=`), 묻지도 따지지도 말고 내 방(중괄호 `{ }`) 안으로 들어와서 죽음 처리를 해! 그 외의 상황이면(`else`) 전부 다 밑에 있는 저 녀석 방으로 쫓아내서 생존 처리를 시켜라!"*

### 📅 [Day 2] switch (선택문) - 100층 직통 로켓 엘리베이터

#### 📖 1. 예습 (로켓 직통 엘리베이터 비유)
> **🚀 `if` 문의 함정과 `switch` 문의 구원**
> 
> 유니티에 무기가 100종류 있습니다. 이걸 멍청하게 `if` 문으로 짜면 기도가 1층부터 100층까지 문을 100번 다 두드리며 물어봅니다. "손님 칼이세요? 아뇨", "그럼 총이세요? 아뇨", "그럼 활이세요? 아뇨..." 100번째 무기를 찾을 때 엄청난 버벅거림(프레임 저하)이 발생합니다.
> 
> 이럴 때 쓰는 것이 **`switch`** 입니다. 얘는 1부터 순서대로 검사하지 않고, 그냥 건물 1층에서 "저 총이요" 라고 말하는 순간 **총 층수(Case)로 중간 과정을 싹 무시하고 0.001초 만에 냅다 꽂아버리는 로켓 엘리베이터**입니다. 메뉴판(Day 2 열거형 Enum)과 궁합이 환상적입니다!

#### ✅ 2. 모범 답안과 독해
```csharp
public enum WeaponType { Sword, Bow, Gun } // [복습] 키오스크 전용 메뉴판!

public void Attack(WeaponType currentWeapon) 
{
    // [문장 1 독해] 100층 로켓 엘리베이터 가동
    switch (currentWeapon) 
    {
        case WeaponType.Sword:
            Debug.Log("칼로 슥삭!");
            break; // [문장 2 독해] 일 끝났으면 탈출!

        case WeaponType.Bow:
            Debug.Log("활로 피슝!");
            break;
            
        case WeaponType.Gun:
            Debug.Log("총으로 탕탕!");
            break;
    }
}
```

**🔍 잠깐! 코드 외계어 독해 가이드**
*   👉 **코드 공식:** `switch (검사할종이) { case 정답1: 행동; break; ... }`
*   🗣️ **소리내어 읽기:** *"지금 내 손에 들린 무기 종류 종이(`currentWeapon`)를 로켓 엘리베이터(`switch`) 버튼에 딱 올려! 만약 그 무기가 칼(`case WeaponType.Sword:`) 버튼이면 그 층에 바로 꽂은 다음 칼부림 액션을 실행해. 다 했으면 엘리베이터 줄 끊고 무조건 밖으로 탈출해(`break`)! 더 이상 밑에 층 검사할 필요 없으니까!"* 
*   **❓ `break` 를 안 쓰면 어떻게 되나요?**
    *   마치 엘리베이터 브레이크가 고장 난 것과 같습니다. 칼 층에서 멈추지 않고 그대로 바닥을 뚫고 내려가서 활도 쏘고 총도 쏘는 대참사가 벌어집니다!

### 📅 [Day 3] for (반복문) - 컨베이어 벨트 공장장

#### 📖 1. 예습 (정확한 횟수 계약 알바생)
> 🏗️ **끝이 정해진 안전한 반복**
> 
> 스테이지에 몬스터를 10번 소환하고 싶습니다. `SpawnMonster()` 함수를 무식하게 10줄 복붙하면 AI입니다.
> 우리 아트 디렉터는 **컨베이어 벨트(`for` 문)**를 한 대 설치하고 알바생에게 명령합니다. "0부터 시작해서 카운트 세고, 정확히 10번 찍으면 기계 끄고 퇴근해!"

#### ✅ 2. 모범 답안과 독해
```csharp
public void SpawnMonsterWave() 
{
    // [문장 1 독해] 10번 돌아가는 컨베이어 벨트
    for (int i = 0; i < 10; i++) 
    {
        SpawnMonster(); 
    }
}
```

**🔍 잠깐! 코드 외계어 독해 가이드**
*   👉 **코드 공식:** `for (시작점; 끝점; 속도) { 공장돌리기 }`
*   🗣️ **소리내어 읽기:** *"이 컨베이어 벨트(`for`) 사이클을 시작할 건데, 카운터 기계 숫자(`int i`)는 맨 처음 `0`부터 출발해. 이 숫자가 `10`보다 작을 동안(`i < 10`)만 이 방에 들어와서 몬스터 소환 기계를 작동시키고, 방을 한 번 나갈 때마다 카운터 숫자를 1씩 꼬박꼬박 올려라(`i++`). 카운터가 10이 되는 순간 공장 셔터 문 닫고 퇴근!"*

### 📅 [Day 4] while (반복문) - 무한 루프의 공포 (게임 프리징)

#### 📖 1. 예습 (악덕 사장의 영원한 굴레)
> 💀 **"조건이 맞을 때까지 계속해!" 그리고 과로사...**
> 
> `for`문이 끝이 정해진 깔끔한 계약직 알바라면, **`while`문은 끝이 안 정해진 무서운 악덕 사장**입니다. "바구니에 사과가 다 비워질 때까지 무한정 일해!" 라는 명령입니다. 
> 만약 지휘관님이 바구니 맨 밑바닥에 사과를 무한히 뿜어내는 호스를 숨겨뒀다면 어떻게 될까요? 불쌍한 알바생(컴퓨터CPU)은 영원히 사과를 치우려고 발악하다가 **뇌가 터져서 과로사**합니다. 
> 유니티 게임을 플레이하다가 허공에서 갑자기 삑! 멈추고 '응답 없음'이 뜨며 컴퓨터 자체가 먹통이 되는 **가장 끔찍한 버그 1순위가 바로 이 무한 루프(Infinite Loop)**의 저주입니다.

#### 🚨 2. 문제와 함정 (게임이 터지는 순간)
```csharp
public void BadWhileLoop() 
{
    int apples = 10;

    // 만약 apples를 깎는 코드를 까먹고 안 썼다면?!
    while (apples > 0) 
    {
        Debug.Log("사과를 먹는다! (영원히...)");
        // 이 안에 apples -= 1; 을 실수로 안 쓰면 사과가 영원히 10개로 고정됨!
        // 컴퓨터는 이 굴레에서 1초에 1억 번을 맴돌다가 응답 없음으로 죽는다.
    }
}
```

#### 🗣️ 3. 지휘관의 프롬프팅
> **"AI야, `while` 쓸 때 루프 탈출 조건(안전장치) 안 걸어두면 무한 루프 돌아서 렉 걸리고 게임 터지는 거 몰라? `while` 내부에서 확실하게 플래그 숫자가 변해서 `break` 타고 빠져나갈 수 있는지 두 번 검수해!"**

### 📅 [Day 5] 흐름 제어 방탈출 모의고사

방탈출의 시간입니다! 아래 AI의 끔찍한 하드코딩 코드를 보고 속으로 호통을 쳐 주십시오.

> **💣 상황:** 몬스터가 죽으면 파티클 5개를 흩뿌리고, 불속성인지 물속성인지 판별하는 로직을 맡겼습니다.

public void OnMonsterDie(string monType) 
{

2. "몬스터 속성을 멍청하게 `if`로 100번 검사해서 CPU 속도 다 까먹을 셈이야? 100층 찔러보지 말고 한방에 물/불 로켓 엘리베이터로 꽂아버리는 `____`(2) 문법 써!"
3. "(만약 `while`을 썼다면) 야! `while`문 썼으면 안에 빠져나갈 구멍 팠는지 확인해. 탈출 조건 안 만들면 알바생이 영원히 일하다가 컴퓨터 과로사하는 `____ ____`(3) 버그 터지잖아!"
